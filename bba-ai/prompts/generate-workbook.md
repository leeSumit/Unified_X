## What to Generate

Produce the full workbook content as markdown text, section by section. A designer will take this text and lay it into the visual template. You are writing all the words, not designing the layout — but structure your output clearly so the designer knows what goes on each page.

Use this labelling:
- `[PAGE BREAK]` — where a new page starts
- `[CHIP: Boot Camp 1]` — for the coloured header chip
- `[LEFT COLUMN]` / `[RIGHT COLUMN]` — for two-column layouts
- `[CARD: title]` — for framework cards
- `[RESPONSE BOX]` — for areas where students write
- `[TABLE: description]` — for tables

## Tone Rules for Workbook Text

- All instructions to students use "you" — direct second person
- Activity headers are active and imperative: "Analyse this", "Build your", "Test against"
- Scenario names use Indian brands, Indian cities, Indian cultural context
- Avoid: "In this activity…", "Students will…", passive voice
- Callout boxes use present tense, authoritative voice

---

## Output Length

The workbook content should cover approximately 35 pages when designed. Your markdown output will be longer because it includes all markup labels. Write complete, publish-ready text for every section — no placeholders, no "[insert content here]" gaps.
