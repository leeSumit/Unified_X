# Workbook Structure Guide

This document defines the exact structure, section patterns, and visual language of every Campus AI student workbook. All workbooks follow the Module 1 blueprint.

---

## Overall Session Architecture

Every module maps to **5 classroom sessions** following this exact sequence:

| Session | Content | Duration |
|---------|---------|---------|
| Session 1 | Boot Camp 1 (Foundations) | 2 hours |
| Session 2 | Project 1 + Case Study 1 (India) | 2 hours |
| Session 3 | Boot Camp 2 (Frameworks/Core Tools) | 2 hours |
| Session 4 | Project 2 + Case Study 2 (Global) | 2 hours |
| Session 5 | Capstone | 2 hours |

---

## Page-by-Page Blueprint

### Cover Page
- Brand: "Campus AI" logo top-left
- Headline: "A student guide to **mastering [module topic] with Campus AI**"
- Subtitle: "STUDENT WORKBOOK"
- Footer: "BBA in Artificial Intelligence · Semester X · Module X · [Module Title] · [N] Contact Hours · Infinite Solutions"
- Abstract geometric illustration (purple/lavender tones)

### Ownership Page
Fields: Student Name, Enrollment No., Section/Batch, Faculty, Academic Year
+ "HOW TO USE THIS" box with 5 numbered instructions

### Index Page (Page 1)
- Title: "Your [N]-hour journey through Module [X]"
- Subtitle italic: "Every page in this workbook maps to one of the five classroom sessions."
- 5-column session map (coloured labels: SESSION 1–5 with page ranges)
- Full table of contents with purple page numbers

### Introduction Page (Page 2)
- Header chip: "INTRODUCTION"
- Title: "Module [X] — [Module Title]"
- Italic subtitle: "[N] hours · Five sessions · One professional skill you will carry for a career."
- Opening paragraph (2-3 sentences) — what the module is about
- Two-column layout:
  - Left: **Learning Outcomes** (LO1–LO4 with Bloom's verb: Remember/Understand/Apply/Analyse)
  - Right: **Course Outcomes** (CO1–CO4 — what students walk out with)
- Left: **Before you begin** (checklist: equipment, prerequisites, pre-reading)
- Right: **AI tools used in this module** (3-4 tools with one-line description each)

### Pedagogy Page (Page 3)
- Header chip: "OUR TEACHING PHILOSOPHY"
- Title: "The Campus AI Pedagogy"
- Italic subtitle: "Why your classroom time is structured the way it is."
- Opening 2-sentence paragraph
- 3-column grid:
  1. **Design Thinking** (IDEO/Stanford d.school) — how it applies to THIS module's hands-on work
  2. **Experiential Learning** (Kolb's cycle, 1984) — the try-reflect-conceptualise-try loop
  3. **Industry-First Content** (No framework without a company) — 3 specific companies used in this module
- "The big picture" callout box (1 paragraph — big-picture framing)

## Boot Camp Sections

Each Boot Camp follows this pattern:

### Boot Camp Opening (1 page)
- Header chip: "BOOT CAMP [N]"
- Title: "[Boot Camp Title]"
- Italic subtitle: "Session [N] · Two hours · [3 comma-separated topics]."
- 2-paragraph introduction (why these topics matter, how they connect)
- **Core content in card format** — paired left/right boxes:
  - Card title (bold purple)
  - Card subtitle (italic smaller)
  - Body text
  - Sub-sections: COMPONENTS, WHEN TO USE, MINI EXAMPLE (in monospace/code box)

### Boot Camp Continuation (1–2 pages)
- Remaining framework cards
- "Combining frameworks is where craft begins" section (if Boot Camp 2)
- **Failure taxonomy** (if relevant to module)
- "Reflect before you leave class" prompt (1 open response box)

---

## Project Sections

### Project Opening (1 page)
- Header chip: "PROJECT [N]" (orange chip, vs purple for Boot Camp)
- Title: "[Project Name]"
- Italic subtitle: "Session [N] · [N] minutes in class · [One-line description]."
- 2-paragraph briefing
- Two-column layout:
  - Left: **Your mission** + **Concepts applied** + **Deliverable**
  - Right: **How to run it** (Step 1–5 numbered instructions)
- **Scenarios list** (5–7 bullet scenarios to choose from)

### Project Worksheet (1–2 pages)
- Tables, matrices, or structured response boxes
- Example structures:
  - Comparison matrix (tools vs criteria, scored 1–5)
  - Sprint challenge cards (4 challenges in 2×2 grid)
  - Response boxes for written work (lined areas)
- "Your three insights" / "Your recommendation" open-response sections

## Case Study Sections

Each module has 2 case studies: **one Indian company** (Session 2) and **one global company** (Session 4).

### Case Study Opening (1 page)
- Header chip: "CASE STUDY [N] · [INDIA/GLOBAL]"
- Title: "[Company]'s [AI Initiative]"
- Italic subtitle: "[Punchy one-liner capturing the arc of the story]."
- 1-paragraph hook (the surprising opening fact or reversal)
- **4-metric dashboard**: 4 stats in large coloured text (purple + orange alternating)
  - Example: $838.8M FY2025 revenue | $25M+ Freddy AI ARR | 75K+ Customers | 45% AI-resolved
- **The company** section (2 paragraphs: origin story + current scale)

### Case Study Body (2–3 pages)
- **The [transformation/deployment/strategy]** section — the main AI story
  - Named people, specific dates, specific decisions
  - Real quotes in italic blockquotes
- **Results/Evidence** section — table or bullet list with specific outcomes per customer/use-case
- **The [contrast/reversal/lesson]** section — the nuanced reading
  - For India case: comparison to a global benchmark
  - For global case: the failure, reversal, or unexpected outcome

### Case Study Discussion Questions (1 page)
- Title: "Discussion questions"
- 2-sentence intro about how to use them
- 4 numbered discussion questions (Q1–Q4)
  - Each question in a rounded box with coloured number indicator
  - Large blank response area below each question
  - Questions follow escalating difficulty: recall → analysis → judgment → ethics/personal stance

## Capstone Section

The capstone is a **2-hour integrative team project** that applies everything from the module.

### Capstone Brief (1 page)
- Header chip: "CAPSTONE"
- Title: "The [Module-Specific Project Title]"
- Italic subtitle: "Session 5 · Two hours · [One-line description]."
- Opening: "For the next two hours you are not a student. You are a [role] brought in to [mission]."
- **The scenario** (company name, context, problem, stakes) — 3–4 paragraphs
- **What you will deliver** — the output + evaluation rubric preview

### Capstone Diagnosis/Analysis Page (1 page)
- The raw material for analysis (broken prompts table / dataset / case documents)
- Students classify/analyse before fixing

### Design Thinking Stages (5 pages, one per stage)
Each stage: header, purpose, "Your task in this stage" callout, large "Your work:" response area

1. **EMPATHISE** — understand the affected user/stakeholder
2. **DEFINE** — frame the problem as "How might we..."
3. **IDEATE** — generate solutions using frameworks
4. **PROTOTYPE** — actually run/test the solutions
5. **TEST** — design A/B test / rollout plan

### Turnaround Deck Structure (1 page)
- Table: 8-slide structure with slide number, slide title, and purpose
- **Evaluation rubric** — 5 criteria with percentage weights

## Reflection Section

### Reflection Page (1 page)
- Header chip: "REFLECTION"
- Title: "What you built. Who you are becoming."
- Italic subtitle: "Module [N] close · Ten minutes of honest writing."
- **Before and after** table (2 columns: BEFORE Module [N] / AFTER Module [N])
- **Your portfolio** — table: Your prompt | Framework(s) used | What it demonstrates

### Career Pathway Page (1 page)
- Salary/career data section (India-specific, from named sources)
- "In one paragraph, answer this question" open-response box
- **Sign-off** section (Student signature / Faculty signature / Date)

---

## Crossword/Puzzle Page

- Title: "The Graduation Puzzle"
- Subtitle: "Ten interlocking terms from everything you learned this module."
- 2-sentence intro
- Crossword grid (visual)
- ACROSS and DOWN clues (10 terms total)
- Answer key on facing page
